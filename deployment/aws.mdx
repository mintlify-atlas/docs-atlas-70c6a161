Deploy your OpenComponents registry to AWS using S3 for component storage. This guide covers S3 configuration, IAM setup, and deployment patterns for EC2, ECS, and Lambda.

## S3 Storage Configuration

OpenComponents includes built-in support for S3 storage via the `oc-s3-storage-adapter` package.

### Basic S3 Configuration

The simplest way to configure S3 storage:

```typescript
const oc = require('oc');

const registry = oc.Registry({
  baseUrl: 'https://components.mycompany.com/',
  s3: {
    bucket: 'my-components-bucket',
    region: 'us-east-1',
    componentsDir: 'components'
  }
});
```

<Note>
When using the `s3` configuration option, the registry automatically creates an S3 storage adapter using the `oc-s3-storage-adapter` package.
</Note>

### S3 Configuration Options

From `src/types.ts:372-378`:

```typescript
s3?: {
  bucket: string;        // S3 bucket name
  region: string;        // AWS region (e.g., 'us-east-1')
  key?: string;          // AWS access key ID (optional with IAM)
  secret?: string;       // AWS secret access key (optional with IAM)
  componentsDir: string; // Directory path within bucket
}
```

### Authentication Methods

<Tabs>
  <Tab title="IAM Roles (Recommended)">
    Use IAM roles when deploying to EC2, ECS, EKS, or Lambda:
    
    ```typescript
    const registry = oc.Registry({
      s3: {
        bucket: 'my-components-bucket',
        region: 'us-east-1',
        componentsDir: 'components'
        // No key/secret - uses IAM role automatically
      }
    });
    ```
    
    <Tip>
    IAM roles are more secure than access keys and rotate credentials automatically.
    </Tip>
  </Tab>
  
  <Tab title="Access Keys">
    Use access keys for local development or non-AWS environments:
    
    ```typescript
    const registry = oc.Registry({
      s3: {
        bucket: 'my-components-bucket',
        region: 'us-east-1',
        key: process.env.AWS_ACCESS_KEY_ID,
        secret: process.env.AWS_SECRET_ACCESS_KEY,
        componentsDir: 'components'
      }
    });
    ```
    
    <Warning>
    Never commit access keys to version control. Use environment variables.
    </Warning>
  </Tab>
  
  <Tab title="Environment Variables">
    The AWS SDK automatically uses standard environment variables:
    
    ```bash
    export AWS_ACCESS_KEY_ID=your-key-id
    export AWS_SECRET_ACCESS_KEY=your-secret-key
    export AWS_REGION=us-east-1
    ```
    
    ```typescript
    const registry = oc.Registry({
      s3: {
        bucket: 'my-components-bucket',
        region: process.env.AWS_REGION,
        componentsDir: 'components'
      }
    });
    ```
  </Tab>
</Tabs>

## IAM Configuration

### Required S3 Permissions

Create an IAM policy with minimum required permissions:

```json iam-policy.json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "OpenComponentsS3Access",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-components-bucket",
        "arn:aws:s3:::my-components-bucket/*"
      ]
    }
  ]
}
```

<Steps>
  <Step title="Create IAM Policy">
    ```bash
    aws iam create-policy \
      --policy-name OpenComponentsS3Policy \
      --policy-document file://iam-policy.json
    ```
  </Step>
  
  <Step title="Create IAM Role (for EC2/ECS)">
    ```bash
    # For EC2
    aws iam create-role \
      --role-name OpenComponentsEC2Role \
      --assume-role-policy-document file://ec2-trust-policy.json
    
    # Attach policy to role
    aws iam attach-role-policy \
      --role-name OpenComponentsEC2Role \
      --policy-arn arn:aws:iam::ACCOUNT_ID:policy/OpenComponentsS3Policy
    ```
  </Step>
  
  <Step title="Create Instance Profile (for EC2)">
    ```bash
    aws iam create-instance-profile \
      --instance-profile-name OpenComponentsProfile
    
    aws iam add-role-to-instance-profile \
      --instance-profile-name OpenComponentsProfile \
      --role-name OpenComponentsEC2Role
    ```
  </Step>
</Steps>

### EC2 Trust Policy

```json ec2-trust-policy.json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "ec2.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

## S3 Bucket Setup

<Steps>
  <Step title="Create S3 Bucket">
    ```bash
    aws s3 mb s3://my-components-bucket --region us-east-1
    ```
  </Step>
  
  <Step title="Configure CORS (if serving to browsers)">
    ```json cors-config.json
    [
      {
        "AllowedOrigins": ["https://myapp.com"],
        "AllowedMethods": ["GET", "HEAD"],
        "AllowedHeaders": ["*"],
        "MaxAgeSeconds": 3000
      }
    ]
    ```
    
    ```bash
    aws s3api put-bucket-cors \
      --bucket my-components-bucket \
      --cors-configuration file://cors-config.json
    ```
  </Step>
  
  <Step title="Enable Versioning (recommended)">
    ```bash
    aws s3api put-bucket-versioning \
      --bucket my-components-bucket \
      --versioning-configuration Status=Enabled
    ```
  </Step>
  
  <Step title="Configure Lifecycle Rules (optional)">
    Automatically delete old component versions:
    
    ```json lifecycle-policy.json
    {
      "Rules": [
        {
          "Id": "DeleteOldVersions",
          "Status": "Enabled",
          "NoncurrentVersionExpiration": {
            "NoncurrentDays": 90
          }
        }
      ]
    }
    ```
    
    ```bash
    aws s3api put-bucket-lifecycle-configuration \
      --bucket my-components-bucket \
      --lifecycle-configuration file://lifecycle-policy.json
    ```
  </Step>
</Steps>

## Deployment Patterns

### EC2 Deployment

<Steps>
  <Step title="Launch EC2 Instance">
    ```bash
    aws ec2 run-instances \
      --image-id ami-0c55b159cbfafe1f0 \
      --instance-type t3.medium \
      --iam-instance-profile Name=OpenComponentsProfile \
      --key-name my-key-pair \
      --security-group-ids sg-xxxxx \
      --subnet-id subnet-xxxxx
    ```
  </Step>
  
  <Step title="Install Node.js and Dependencies">
    ```bash
    # SSH into instance
    ssh -i my-key-pair.pem ec2-user@instance-ip
    
    # Install Node.js (v18+)
    curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
    sudo yum install -y nodejs
    
    # Clone your registry code
    git clone https://github.com/mycompany/oc-registry.git
    cd oc-registry
    npm install
    ```
  </Step>
  
  <Step title="Configure and Start Registry">
    ```bash
    # Set environment variables
    export BASE_URL=https://components.mycompany.com/
    export S3_BUCKET=my-components-bucket
    export AWS_REGION=us-east-1
    
    # Start registry with PM2
    npm install -g pm2
    pm2 start registry.js
    pm2 save
    pm2 startup
    ```
  </Step>
  
  <Step title="Configure Load Balancer">
    Create an Application Load Balancer:
    
    - Target: EC2 instance on port 3000
    - Health check: `GET /` or custom health endpoint
    - SSL/TLS certificate from ACM
    - Security groups: Allow HTTPS (443) from internet
  </Step>
</Steps>

### ECS Deployment

<Steps>
  <Step title="Create ECS Task Definition">
    ```json task-definition.json
    {
      "family": "oc-registry",
      "taskRoleArn": "arn:aws:iam::ACCOUNT_ID:role/OpenComponentsECSRole",
      "executionRoleArn": "arn:aws:iam::ACCOUNT_ID:role/ecsTaskExecutionRole",
      "networkMode": "awsvpc",
      "containerDefinitions": [
        {
          "name": "oc-registry",
          "image": "mycompany/oc-registry:latest",
          "portMappings": [
            {
              "containerPort": 3000,
              "protocol": "tcp"
            }
          ],
          "environment": [
            {
              "name": "BASE_URL",
              "value": "https://components.mycompany.com/"
            },
            {
              "name": "S3_BUCKET",
              "value": "my-components-bucket"
            },
            {
              "name": "AWS_REGION",
              "value": "us-east-1"
            }
          ],
          "logConfiguration": {
            "logDriver": "awslogs",
            "options": {
              "awslogs-group": "/ecs/oc-registry",
              "awslogs-region": "us-east-1",
              "awslogs-stream-prefix": "ecs"
            }
          }
        }
      ],
      "requiresCompatibilities": ["FARGATE"],
      "cpu": "512",
      "memory": "1024"
    }
    ```
  </Step>
  
  <Step title="Create ECS Service">
    ```bash
    aws ecs create-service \
      --cluster my-cluster \
      --service-name oc-registry \
      --task-definition oc-registry \
      --desired-count 2 \
      --launch-type FARGATE \
      --network-configuration "awsvpcConfiguration={subnets=[subnet-xxxxx],securityGroups=[sg-xxxxx],assignPublicIp=ENABLED}" \
      --load-balancers "targetGroupArn=arn:aws:elasticloadbalancing:...,containerName=oc-registry,containerPort=3000"
    ```
  </Step>
</Steps>

### Lambda Deployment

<Warning>
Lambda deployment has limitations due to cold starts and execution time limits. Best for low-traffic registries.
</Warning>

Use the Serverless Framework:

```yaml serverless.yml
service: oc-registry

provider:
  name: aws
  runtime: nodejs18.x
  region: us-east-1
  iamRoleStatements:
    - Effect: Allow
      Action:
        - s3:GetObject
        - s3:PutObject
        - s3:ListBucket
      Resource:
        - arn:aws:s3:::my-components-bucket
        - arn:aws:s3:::my-components-bucket/*
  environment:
    S3_BUCKET: my-components-bucket
    AWS_REGION: us-east-1
    BASE_URL: https://components.mycompany.com/

functions:
  registry:
    handler: lambda.handler
    timeout: 30
    memorySize: 1024
    events:
      - http:
          path: /{proxy+}
          method: ANY
```

## Advanced Storage Configuration

### Custom Storage Adapter

For advanced use cases, use the storage adapter directly:

```typescript
const s3Adapter = require('oc-s3-storage-adapter');

const registry = oc.Registry({
  baseUrl: 'https://components.mycompany.com/',
  storage: {
    adapter: s3Adapter,
    options: {
      bucket: 'my-components-bucket',
      region: 'us-east-1',
      componentsDir: 'components',
      
      // Advanced options
      refreshInterval: 60,  // Cache refresh interval
      verbosity: 1          // Enable logging
    }
  }
});
```

### S3-Compatible Storage

Use S3-compatible services like MinIO, DigitalOcean Spaces, or Wasabi:

```typescript
const s3Adapter = require('oc-s3-storage-adapter');

const registry = oc.Registry({
  storage: {
    adapter: s3Adapter,
    options: {
      bucket: 'my-components',
      region: 'us-east-1',
      componentsDir: 'components',
      endpoint: 'https://nyc3.digitaloceanspaces.com', // Custom endpoint
      key: process.env.SPACES_KEY,
      secret: process.env.SPACES_SECRET
    }
  }
});
```

## Monitoring

<Tabs>
  <Tab title="CloudWatch Metrics">
    Monitor S3 and registry metrics:
    
    ```typescript
    const AWS = require('aws-sdk');
    const cloudwatch = new AWS.CloudWatch();
    
    registry.on('request', (data) => {
      // Send custom metrics to CloudWatch
      cloudwatch.putMetricData({
        Namespace: 'OpenComponents',
        MetricData: [
          {
            MetricName: 'ComponentRenders',
            Value: 1,
            Unit: 'Count'
          }
        ]
      });
    });
    ```
  </Tab>
  
  <Tab title="CloudWatch Logs">
    Stream logs to CloudWatch:
    
    ```javascript
    const winston = require('winston');
    const CloudWatchTransport = require('winston-cloudwatch');
    
    const logger = winston.createLogger({
      transports: [
        new CloudWatchTransport({
          logGroupName: '/oc/registry',
          logStreamName: 'production',
          awsRegion: 'us-east-1'
        })
      ]
    });
    
    registry.on('error', (error) => {
      logger.error('Registry error', error);
    });
    ```
  </Tab>
  
  <Tab title="X-Ray Tracing">
    Enable AWS X-Ray for distributed tracing:
    
    ```javascript
    const AWSXRay = require('aws-xray-sdk');
    const AWS = AWSXRay.captureAWS(require('aws-sdk'));
    
    // X-Ray will automatically trace S3 calls
    ```
  </Tab>
</Tabs>
