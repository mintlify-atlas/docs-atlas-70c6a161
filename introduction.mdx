## Welcome to OpenComponents

OpenComponents is an open-source framework that allows fast-moving teams to easily build and deploy front-end components. It abstracts away complicated infrastructure and leaves developers with very simple, but powerful building blocks that handle scale transparently.

## How It Works

OpenComponents follows a simple three-step workflow:

<Steps>
  <Step title="Create Your Component">
    Build components with server-side logic using Node.js and client-side views using your favorite framework (React, Vue, or plain JavaScript). Include CSS and JavaScript - it's all self-contained.

    ```bash
    oc init my-component oc-template-es6
    ```
  </Step>

  <Step title="Publish to Registry">
    Publish your component to an OpenComponents registry. The registry automatically prepares your component to be production-ready, optimizing assets and handling versioning.

    ```bash
    oc publish my-component
    ```
  </Step>

  <Step title="Consume Anywhere">
    Every component gets its own HTTP endpoint. Consume it during server-side rendering or directly in the browser from any web application in your network.

    ```html
    <oc-component href="http://my-registry.com/my-component/1.0.0"></oc-component>
    ```
  </Step>
</Steps>

## Why OpenComponents?

<Info>
  OpenComponents has been battle-tested in production environments, serving millions of requests daily. It provides the infrastructure for distributed front-end development without the complexity.
</Info>

OpenComponents bridges the gap between microservices architecture and front-end development. Just as microservices allow backend teams to deploy independently, OpenComponents enables front-end teams to:

- **Deploy independently** - Update components without redeploying consuming applications
- **Scale independently** - Components are served from a registry with built-in caching and CDN support
- **Version independently** - Multiple versions coexist, allowing gradual rollouts
- **Develop independently** - Teams own their components end-to-end

## Requirements

<Note>
  OpenComponents requires **Node.js 18 or higher**
</Note>

The framework is designed to work with modern Node.js versions and supports both CommonJS and ES modules.
