## Overview

Static assets are files like stylesheets, images, fonts, and client-side JavaScript that your component needs. OpenComponents handles packaging, CDN distribution, and cache invalidation for these files automatically.

## Adding Static Files

### Directory Structure

Create a directory for your static assets (commonly named `static/`, `files/`, or `assets/`):

```
my-component/
├── package.json
├── server.js
├── template.html
└── static/
    ├── styles.css
    ├── logo.png
    └── script.js
```

### Configuration

Reference static files in `package.json`:

```json
{
  "name": "my-component",
  "version": "1.0.0",
  "oc": {
    "files": {
      "template": {
        "src": "template.html",
        "type": "handlebars"
      },
      "static": ["static/"]
    }
  }
}
```

<Info>
You can specify:
- A directory: `"static/"` (includes all files in the directory)
- Individual files: `"logo.png"`, `"styles.css"`
- Multiple paths: `["static/", "images/logo.png"]`
</Info>

### Individual Files

```json
{
  "oc": {
    "files": {
      "template": {
        "src": "template.html",
        "type": "handlebars"
      },
      "static": [
        "styles.css",
        "logo.png",
        "script.js"
      ]
    }
  }
}
```

## Accessing Static Assets

### In Templates

Use `context.staticPath` to get the base URL for your static assets:

**Handlebars:**
```handlebars
<link rel="stylesheet" href="{{staticPath}}styles.css" />
<img src="{{staticPath}}logo.png" alt="Logo" />
<script src="{{staticPath}}script.js"></script>
```

**Jade:**
```jade
link(rel="stylesheet", href=staticPath + 'styles.css')
img(src=staticPath + 'logo.png', alt="Logo")
script(src=staticPath + 'script.js')
```

**ES6:**
```javascript
export default function(props) {
  return (
    <div>
      <link rel="stylesheet" href={props.staticPath + 'styles.css'} />
      <img src={props.staticPath + 'logo.png'} alt="Logo" />
    </div>
  );
}
```

### In Data Provider

Pass asset URLs to your template:

```javascript
module.exports.data = function(context, callback) {
  callback(null, {
    logoUrl: context.staticPath + 'logo.png',
    avatarUrl: context.staticPath + 'avatars/user.jpg',
    styleUrl: context.staticPath + 'theme.css'
  });
};
```

**In template:**
```handlebars
<img src="{{logoUrl}}" alt="Logo" />
<link rel="stylesheet" href="{{styleUrl}}" />
```

## Static Path URLs

The `staticPath` is automatically generated by the registry and includes:

- Component name
- Component version
- Hash key for cache busting

**Example:**
```
https://registry.com/components/my-component/1.0.0/static/
```

This ensures:
- **Unique URLs** for different versions
- **Cache invalidation** when assets change
- **CDN compatibility** for fast delivery

## Jade Include Filters

Jade/Pug templates have powerful file inclusion with automatic minification:

### CSS Filters

**include:uglify-css**
```jade
style
  include:uglify-css ./files/styles.css
```

**include:clean-css**
```jade
style
  include:clean-css ./files/theme.css
```

Both filters inline and minify CSS. The difference:
- `uglify-css`: Basic minification
- `clean-css`: Advanced optimization with restructuring

### JavaScript Filters

**include:uglify-js**
```jade
script
  include:uglify-js ./files/analytics.js
```

Inlines and minifies JavaScript code.

### Why Use Filters?

<AccordionGroup>
  <Accordion title="Performance">
    Inlining critical CSS/JS eliminates additional HTTP requests, improving initial page load.
  </Accordion>
  
  <Accordion title="Simplicity">
    No build step needed for basic asset optimization.
  </Accordion>
  
  <Accordion title="Atomic Components">
    Everything needed for the component is packaged together.
  </Accordion>
</AccordionGroup>

<Warning>
Only use filters for small, critical assets. Large files should be loaded as external resources to take advantage of browser caching.
</Warning>

## CSS Stylesheets

### External Stylesheet

**styles.css:**
```css
.my-component {
  padding: 20px;
  background: #f5f5f5;
  border-radius: 8px;
}

.my-component h1 {
  color: #333;
  font-size: 24px;
}
```

**template.html:**
```handlebars
<link rel="stylesheet" href="{{staticPath}}styles.css" />
<div class="my-component">
  <h1>{{title}}</h1>
</div>
```

### Inline Styles (Jade)

```jade
style
  include:clean-css ./files/critical.css

.my-component
  h1= title
```

### Scoped Styles

Prefix your CSS classes with the component name to avoid conflicts:

```css
/* Good: Scoped to component */
.user-profile { }
.user-profile__avatar { }
.user-profile__name { }

/* Bad: Global styles that may conflict */
.avatar { }
.name { }
```

## Images

### Static Images

**package.json:**
```json
{
  "oc": {
    "files": {
      "static": ["static/"]
    }
  }
}
```

**Directory:**
```
static/
├── logo.png
├── icon-success.svg
└── background.jpg
```

**template.html:**
```handlebars
<div class="header">
  <img src="{{staticPath}}logo.png" alt="Company Logo" />
</div>

<div class="message success">
  <img src="{{staticPath}}icon-success.svg" alt="" />
  <p>{{message}}</p>
</div>
```

### Dynamic Images

Pass image URLs from data provider:

```javascript
module.exports.data = function(context, callback) {
  const userId = context.params.userId;
  
  callback(null, {
    avatarUrl: context.staticPath + `avatars/${userId}.jpg`,
    defaultAvatar: context.staticPath + 'default-avatar.png'
  });
};
```

**template:**
```handlebars
<img src="{{avatarUrl}}" alt="User Avatar" 
     onerror="this.src='{{defaultAvatar}}'" />
```

### Responsive Images

```handlebars
<picture>
  <source media="(min-width: 768px)" 
          srcset="{{staticPath}}hero-large.jpg" />
  <source media="(min-width: 480px)" 
          srcset="{{staticPath}}hero-medium.jpg" />
  <img src="{{staticPath}}hero-small.jpg" alt="Hero" />
</picture>
```

## Client-Side JavaScript

### External Script

**script.js:**
```javascript
(function() {
  'use strict';
  
  // Component initialization
  const component = document.querySelector('.my-component');
  
  if (component) {
    component.addEventListener('click', function() {
      console.log('Component clicked!');
    });
  }
})();
```

**template.html:**
```handlebars
<div class="my-component">
  <button>Click me</button>
</div>
<script src="{{staticPath}}script.js"></script>
```

### Inline Script (Jade)

```jade
.my-component
  button Click me

script
  include:uglify-js ./files/handler.js
```

### Passing Data to JavaScript

Embed data in the HTML for JavaScript to use:

```handlebars
<div class="my-component" 
     data-user-id="{{userId}}"
     data-api-url="{{apiUrl}}">
  <!-- content -->
</div>

<script>
  const component = document.querySelector('.my-component');
  const userId = component.dataset.userId;
  const apiUrl = component.dataset.apiUrl;
  
  fetch(`${apiUrl}/users/${userId}`)
    .then(res => res.json())
    .then(data => console.log(data));
</script>
```

## Fonts

Include web fonts as static assets:

**fonts.css:**
```css
@font-face {
  font-family: 'CustomFont';
  src: url('fonts/custom-font.woff2') format('woff2'),
       url('fonts/custom-font.woff') format('woff');
  font-weight: normal;
  font-style: normal;
}

.my-component {
  font-family: 'CustomFont', sans-serif;
}
```

**Directory:**
```
static/
├── fonts.css
└── fonts/
    ├── custom-font.woff2
    └── custom-font.woff
```

<Warning>
Font paths in CSS are relative to the CSS file location, not the component root.
</Warning>

## Asset Optimization

### During Build

When you run `oc package`:

1. **Static files are copied** to `_package/`
2. **Hash keys are generated** for cache invalidation
3. **Paths are updated** in the compiled template
4. **Jade filters process** and minify included files

### Best Practices

<CardGroup cols={2}>
  <Card title="Optimize Images" icon="image">
    Compress images before adding them. Use tools like ImageOptim, TinyPNG, or sharp.
  </Card>
  
  <Card title="Minify Assets" icon="compress">
    Minify CSS and JS files. Jade filters do this automatically for included files.
  </Card>
  
  <Card title="Use CDN" icon="globe">
    Configure your registry to serve static assets via CDN for global distribution.
  </Card>
  
  <Card title="Cache Effectively" icon="clock">
    Set appropriate Cache-Control headers on the registry for static assets.
  </Card>
</CardGroup>

## Troubleshooting

### Assets Not Loading

**Check package.json:**
```json
{
  "oc": {
    "files": {
      "static": ["static/"]  // Is this configured?
    }
  }
}
```

**Check file paths:**
- Paths are relative to component root
- Ensure files exist before packaging
- Check `.npmignore` isn't excluding them

**Check template syntax:**
```handlebars
<!-- Correct -->
<img src="{{staticPath}}logo.png" />

<!-- Wrong: missing staticPath -->
<img src="logo.png" />
```

### 404 Errors

- **Verify** files are in `_package/` after building
- **Check** the staticPath URL in browser DevTools
- **Ensure** the file name matches exactly (case-sensitive)

### Cache Issues

If assets aren't updating:

1. Increment component version
2. Republish the component
3. Clear browser cache
4. Check CDN cache settings
