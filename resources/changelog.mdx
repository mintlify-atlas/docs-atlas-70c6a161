## TypeScript Migration

### v0.49.2 - TypeScript Refactoring

Starting from v0.49.2, OpenComponents has been progressively migrated to TypeScript:

- Full TypeScript support with strict mode
- Better IDE integration and autocomplete
- Type definitions for all public APIs
- Improved code maintainability

Key PRs:
- [#1216](https://github.com/opencomponents/oc/pull/1216) - Initial TypeScript setup
- [#1240](https://github.com/opencomponents/oc/pull/1240) - Refactored remaining files to TS
- [#1238](https://github.com/opencomponents/oc/pull/1238) - Typed Registry and routes

## Semantic Versioning

OpenComponents follows [Semantic Versioning](https://semver.org/):

- **Patch (0.50.x):** Bug fixes and minor improvements
- **Minor (0.x.0):** New features (breaking changes during 0.x)
- **Major (x.0.0):** Breaking changes (1.0.0 not yet released)

Note: During 0.x releases, minor versions may include breaking changes. Always check the [migration guide](/resources/migration-guide) before upgrading.
